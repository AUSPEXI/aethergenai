Financial Crime Labs: Safe Scenario Testing with Synthetic Graphs

By Gwylym Owen — 14–18 min read

Executive Summary

Fraud and AML teams need to test ideas quickly without exposing customer data. AethergenPlatform generates synthetic transaction graphs that preserve network structure, seasonal flows, and edge‑case behaviors while removing direct identifiers. This enables repeatable experiments, procurement‑grade evaluations, and evidence‑backed deployments.

Graph Data Model

  • Nodes: customers, accounts, cards, merchants, devices, IPs, branches.
  • Edges: transfers/payments with timestamp, amount, channel, MCC, country, auth result.
  • Node/edge features: tenure, KYC tier, merchant risk band, device fingerprint, velocity bands.
  • Global structure: degree distributions, community structure, diurnal/seasonal cycles.

Typology Library (Parameterised)

  • Structuring (smurfing): micro‑deposits under reporting thresholds; window and band parameters.
  • Mule rings: device/IP reuse across identities; control ring size and churn.
  • Card testing: rapid low‑value authorizations across MCCs; burst/cooldown controls.
  • Sanctions evasion motifs: indirect paths via neutral hubs; path length and timing knobs.
  • First‑party abuse: refund/chargeback loops; merchant collusion parameters.

Evaluation That Sticks in Risk Committees

  • Operating point utility: detection at fixed alert budgets with confidence intervals.
  • Segment stability: product, region, and lifecycle consistency.
  • Scenario stress: performance under parameter sweeps for each typology.
  • Cost curves: incremental cases per analyst‑hour; escalation loads.

Modeling Baselines

  • Rules: transparent baselines for governance and anchoring.
  • Graph features: PageRank‑like centrality, motif counts, community scores.
  • Learned models: GNN or gradient‑boosted trees with calibrated thresholds.

Evidence Bundle

  • Signed lineage and environment fingerprints.
  • Metrics with CIs; per‑typology parameter logs.
  • Ablations to show which features/typologies drive lift.
  • Limits, intended use, and rollback rules.

AethergenPlatform lets you ask hard questions safely: What happens to alert yield if we cut our budget 20%? Which motifs crumble first under drift? Answer with evidence, not anecdotes.

Scenario Design

  • Pick 3 typologies and 2 operating budgets (alerts/day).
  • Define success: cases/analyst‑hour uplift vs baseline.
  • Run parameter sweeps and publish sensitivity.

Feature Catalog

  • Velocity bands, burstiness, merchant diversity.
  • Graph motifs: stars, cycles, bi‑cliques.
  • Community leakage scores; device/IP reuse ratios.

Thresholding Policy

  • Segment thresholds by product and region.
  • Cap escalations; elastic budget for weekends/peaks.
  • Auto‑tune to keep alert budget within ±10%.

FAQ

Will synthetic hurt performance on real traffic?

We measure relative rankings stability and de‑risk via shadow evaluation before promotion. Synthetic is for safe iteration and procurement evidence.

Can we export the graphs?

Yes—Parquet/Delta with documented schemas; evidence bundle includes seeds for regeneration.

Glossary

  • Operating budget: fixed number of alerts per time window.
  • Motif: small recurring subgraph structure.
  • Shadow evaluation: parallel scoring without impacting production decisions.
