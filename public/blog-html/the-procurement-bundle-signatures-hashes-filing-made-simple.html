The Procurement Bundle: Signatures, Hashes, and Filing Made Simple

By Gwylym Owen — 45–60 min read

Executive Summary

Procurement does not want mystery. They want files they can keep: metrics, manifests, hashes, SBOMs, and readable dashboards. AethergenPlatform ships a procurement bundle with every release so buyers can file and audit without phone calls.

What’s in the Bundle

  • Evidence: utility@OP, stability, robustness, latency, privacy.
  • Configs: evaluation and thresholds.
  • Dashboards: HTML/PDF exports; no external links required.
  • Manifests: file hashes; environment fingerprints.
  • SBOM: third‑party components and licenses.
  • Release notes: human‑readable summary with manifest IDs.

Manifest Example

{
  "version": "2025.01",
  "artifacts": ["metrics/utility@op.json", "plots/roc_pr.html", "sbom.json"],
  "hashes": {"metrics/utility@op.json": "sha256:..."},
  "seeds": "seeds/seeds.txt",
  "env": {"python": "3.11", "libs": {"numpy": "1.26"}}
}

Signatures

  • Artifacts signed; signature files stored with bundle.
  • Public keys and rotation policy included.
  • Verification instructions in README.

SBOM

  • Component list with versions and licenses.
  • Vulnerability scan report.
  • Attestations attached for offline review.

Dashboards

  • OP utility, stability by segment, and latency distributions.
  • No internet access required; inline assets only.
  • Anchored to manifest IDs for audit trails.

Filing Workflow

  1. Verify signatures and hashes.
  2. Review dashboards and metrics against thresholds.
  3. File SBOM and manifests; archive in evidence store.
  4. Record acceptance with bundle ID.

Governance

  • Bundle IDs referenced in contracts and SOWs.
  • Deprecation policy and migration notes included.
  • Retention timelines and access controls specified.

Case Study

An insurer ran offline review: they verified hashes, opened HTML dashboards, and filed SBOMs. With no external dependencies, procurement finished in days—not weeks.

FAQ

Can we split bundles by audience?

Yes—engineering annex can be separate; procurement bundle stays stable.

What if a dashboard references a missing asset?

We embed assets or bundle them locally; verification is part of CI.

Glossary

Verification Steps (Detailed)

  1. Compute and compare hashes.
  2. Verify signatures with provided public keys.
  3. Open HTML dashboards; confirm OP and stability bands.
  4. Review SBOM licenses and vulnerability report.
  5. Record bundle ID and decision.

Public Keys & Rotation

  • Include current and previous keys; mark active.
  • Rotation schedule documented; revocation list shipped.
