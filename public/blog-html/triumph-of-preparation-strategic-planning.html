Principles

  • Design then build: schemas, constraints, and interfaces first.
  • Validate small: prove patterns at tiny scale before you scale.
  • Write gates: promote only when evidence passes.
  • Document decisions: avoid re‑debating settled topics.

Architecture Discipline

  • Module boundaries and contracts.
  • Source of truth for thresholds and configuration.
  • Versioning and deprecation paths.

Small‑Scale Validation

  • Prototype recipes and pipelines on tiny slices.
  • Measure effect sizes and drift behavior early.
  • Lock seeds and record hashes for reproducibility.

Evidence Gates

  • Utility at OP with CIs.
  • Stability within segment bands.
  • Latency SLOs; privacy probes.

Goals & Constraints

  • Define KPIs at operating points (OPs) with business owners.
  • Budget latency and resource envelopes (p95/p99; device caps).
  • Privacy posture (probes; optional DP budgets) and evidence cadence.

Architecture & Contracts

  • Module inputs/outputs; error handling; idempotency rules.
  • Config tables as source of truth (no hard‑coding thresholds).
  • Deprecation paths and migration windows.

Data Schemas & Vocabularies

entities:
  Patient: {id, age, region}
  Provider: {id, specialty, region}
  Claim: {id, patient_id, provider_id, date, pos, amount}
  LineItem: {id, claim_id, cpt, icd10, units}
constraints:
  amount ≥ 0; units ≥ 1; CPT in CPT_v12
relations:
  Patient 1..* Claim; Claim 1..* LineItem
vocabularies:
  CPT_v12, ICD10_subset (with region overlays)

Pipelines & Artifacts

ingest → normalise → join → validate → package → deploy → evidence
                     ↘ tests & gates ↗
artifacts: parquet/delta, notebooks, dashboards, manifests, sbom

Evidence Gates & Thresholds

gate.utility@op.min = 0.75
gate.stability.region.max_delta = 0.03
gate.latency.p95_ms = 120; p99_ms = 180
gate.privacy.membership_advantage_max = 0.05

Rollback Triggers & SOPs

  • Trigger: gate breach (utility, stability, latency, privacy).
  • Actions: revert to last good bundle; verify OP; open incident.
  • Documentation: snapshot evidence; post‑mortem within 48h.

Security, Privacy, Compliance

  • Signed manifests; key rotation; revocation lists.
  • Privacy probes and optional DP budgets with impact notes.
  • SBOM per release; export control statements (if applicable).

Runbooks & On‑Call

promotion:
  - ensure gates PASS; sign evidence; update change‑control
  - notify stakeholders; update catalog comments with evidence ID
rollback:
  - run revert script; verify OP; attach logs; notify
incident:
  - snapshot; classify; mitigate; file root cause; assign actions

Templates (Copy‑Paste Ready)

  bundle_id: string

Dashboards (Sections)

  • Executive summary: OP, stability, latency, privacy.
  • Segment tables with CIs and deltas.
  • Ablation forest plots and threshold sweeps.
  • Appendix: configs, hashes, environment.

Red‑Team Checks

  • Try to promote with missing dashboards → packaging gate blocks.
  • Tamper thresholds mid‑run → config hash mismatch stops pipeline.
  • Skip stability calculation → gates fail closed by default.

FAQ (Extended)

What if the plan changes?

Version the master doc; record decisions; re‑run validation at small scale before merging.

How do we avoid over‑planning?

Time‑box drafting; validate fast; keep documents concise and living.

Is every gate mandatory?

For production promotions—yes. Experimental branches can skip but cannot deploy.
