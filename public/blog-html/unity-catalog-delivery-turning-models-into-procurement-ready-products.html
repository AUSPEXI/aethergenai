Unity Catalog Delivery: Turning Models into Procurement‑Ready Products

By Gwylym Owen — 30–45 min read

Executive Summary

Unity Catalog is where data and AI assets become governed products. AethergenPlatform packages datasets and models for Unity Catalog with evidence bundles, entitlements, lineage, and change‑control hooks—so buyers can evaluate, adopt, and operate with confidence.

What “Procurement‑Ready” Means

  • Registration: assets exist as catalog objects with descriptions and grants.
  • Evidence: metrics at operating points with confidence intervals, signed and reproducible.
  • Entitlements: access mapped to groups/roles and contractual tiers.
  • Lineage: sources and transformations recorded.
  • Change‑control: versions, manifests, and rollback SOPs.
  • Support: refresh cadence, SLAs, and contact routes.

Asset Types

  • Tables (Delta/Parquet) for datasets and features.
  • Functions for inference (e.g., model UDFs).
  • Models in MLflow/ONNX with registry entries.
  • Views for consumable slices with stability contracts.

Registration SOP

  1. Create catalog and schema (admin).
  2. Register tables/models; apply grants.
  3. Add descriptions linking to evidence manifests.
  4. Attach lineage metadata and tags (tier, RMAs).
  5. Export HTML/PDF cards for procurement files.

Descriptions that Help Buyers

COMMENT ON TABLE prod.ai.claims
IS 'Purpose: fraud triage. OP: fpr=0.01. Evidence: manifest 2025.01. Refresh: monthly.';

COMMENT ON FUNCTION prod.ai.fraud_infer
IS 'Model v2025.01. Threshold=0.73. Stability bands: region<=0.03. Evidence bundle linked.';

Entitlements

  • Self‑Service: read access to public samples and docs.
  • Assisted: read on packaged tables and UDFs; priority support.
  • Full‑Service: private schemas, custom adapters, SLAs.

Evidence Integration

  • Evidence manifest ID listed in object comments.
  • Artifacts saved to governed storage with hashes.
  • Dashboards exported to HTML/PDF alongside cards.

Lineage & Governance

  • Lineage from seeds → generation → validation → packaging → catalog.
  • Change‑logs reference semantic versions and evidence IDs.
  • Incidents and rollbacks attach to artifact hashes.

Operating Points in Catalog

  • Store thresholds in config tables, not code.
  • Expose current OP and alternatives for scenario planning.
  • Publish effect sizes for recent ablations with CIs.

Buyer Journey (Within Catalog)

  1. Discover asset via search or curated schema.
  2. Read description and evidence link.
  3. Open sample notebook; run inference at OP.
  4. Evaluate effect sizes and stability summaries.
  5. Request entitlements; proceed to contract.

Sample Notebook Outline

  • Connect to catalog; confirm grants.
  • Load sample table; run UDF; compute metrics at OP.
  • Plot trade‑off; inspect segment stability.
  • Record results and evidence manifest ID.

Tables that Stay Useful

Security & Compliance

Pricing & Entitlements Hook

FAQ

Can we publish private listings only?

Yes—private schemas and grants align with contractual entitlements; evidence still ships.

How are breaking changes handled?

Deprecation periods with views and migration notes; card and evidence versions advance.

FAQ (More)

Can we embed OP thresholds in UDFs?

Prefer config tables to avoid code redeploys; UDFs read current thresholds.

How do we handle multi‑tenant customers?

Use per‑tenant schemas and tags; meter usage with workspace logs.
